## 2. What YOU Must Install Before Development (in order)

### 2.1 Core toolchain
1. **Git for Windows** — https://git-scm.com/download/win
2. **JDK 17 (Temurin)** — https://adoptium.net → set `JAVA_HOME`, add to `PATH`.
3. **Flutter SDK (stable)** — https://docs.flutter.dev/get-started/install/windows
   - Extract to e.g. `C:\src\flutter`, add `flutter\bin` to `PATH`.
4. **FVM (Flutter Version Management)** — to pin/freeze Flutter for the whole team:
   ```
   dart pub global activate fvm
   fvm install <exact_stable_version>
   fvm use <exact_stable_version>   # commits .fvmrc — this freezes the version (Rule 3)
   ```

### 2.2 Android tooling
5. **Android Studio (latest stable)** — https://developer.android.com/studio
   - During setup, via **SDK Manager** install:
     - Android **SDK Platform 36** (compileSdk) + **SDK Platform 35** (targetSdk)
     - Android **SDK Platform 29 (Android 10)** ← mandatory test target (Rule 1)
     - Android **SDK Platform 26** (older-device testing)
     - **Android SDK Build-Tools 36.x**
     - **Android SDK Command-line Tools**
     - **Android SDK Platform-Tools** (adb)
     - **Android Emulator**
     - **NDK (Side by side) 28.2.13676358** + **CMake 3.22.1** — only needed for native C/C++ builds; safe to defer until a build asks for them
   - Create **AVDs (emulators):** one **Android 10 (API 29)**, one Android 8/9, one Android 14/15. (Also plan for at least one **physical Android 10 device** — emulators don't catch everything.)
6. **Accept Android licenses:** `flutter doctor --android-licenses`
7. **Run `flutter doctor -v`** and resolve every ✗ before proceeding.

### 2.3 Editor & extensions
8. **VS Code** (you're already using it) + extensions:
   - **Flutter** (Dart-Code), **Dart**, **Error Lens**, **GitLens**, **Prettier** (web), **ESLint** (web).

### 2.4 Backend & infra (for the backend team / local full-stack)
9. **Node.js 20 LTS** via **nvm-windows** (https://github.com/coreybutler/nvm-windows):
   ```
   nvm install 20
   nvm use 20          # add .nvmrc with "20" to freeze
   ```
10. **pnpm** (or stick to npm) — `npm i -g pnpm` (pin the version).
11. **Docker Desktop** — https://www.docker.com/products/docker-desktop (enable WSL2 backend).
12. **PostgreSQL client** + **Redis** (run via Docker locally; no need to install natively).
13. **DBeaver** or **TablePlus** — DB GUI.
14. **Postman** or **Insomnia** — API testing.

## 3. First-Day Verification Checklist
- [ ] `flutter doctor -v` shows no ✗ (Android toolchain, licenses, device).
- [ ] `fvm use <version>` done; `.fvmrc` committed → Flutter version frozen.
- [ ] Android 10 (API 29) emulator + one physical device connected and detected (`adb devices`).
- [ ] `java -version` → 17; `node -v` → 20; `docker --version` OK.
- [ ] A throwaway `flutter create` app **builds and runs on the Android 10 emulator** (proves the toolchain + Rule 1 target).
- [ ] Release build test: `flutter build apk --release --split-per-abi` succeeds and installs on Android 10.
- [ ] All lockfiles present and committed (`pubspec.lock`, `package-lock.json`/`pnpm-lock.yaml`, `.nvmrc`, `.fvmrc`, Gradle wrapper).
- [ ] Firebase/Maps/Razorpay **dev** keys obtained and stored in secret manager (not committed).

## 4. Android Gradle Baseline (APPLIED — `android/app/build.gradle.kts`, Kotlin DSL)
```kotlin
android {
    compileSdk = flutter.compileSdkVersion   // Flutter 3.44.5 -> API 36
    ndkVersion = "28.2.13676358"             // pinned (Rule 3)
    defaultConfig {
        minSdk = 24        // Android 7.0 — broad reach (Rule 1)
        targetSdk = 35     // Android 15 — Play 2026 requirement
        ndk { abiFilters += listOf("arm64-v8a", "armeabi-v7a") }
    }
    buildTypes {
        release {
            isMinifyEnabled = true
            isShrinkResources = true
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro",
            )
        }
    }
}
```
Keep rules live in `android/app/proguard-rules.pro`. Release build still uses the **debug** signing config — swap in a real keystore before publishing.
Build/verify this release config **on an Android 10 device** before shipping (Rule 1.5).
